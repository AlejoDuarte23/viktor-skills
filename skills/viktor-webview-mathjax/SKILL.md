# VIKTOR WebView MathJax

Use this skill when a VIKTOR `WebView` renders an HTML report with LaTeX-style equations.

## Workflow

1. Load `../viktor-webview/SKILL.md` first.
2. Return a full HTML document through `vkt.WebResult(html=...)` or `vkt.WebResult.from_path(...)`; do not rely on the parent VIKTOR page to provide CSS, JavaScript, or MathJax.
3. Configure `window.MathJax` before loading the MathJax script.
4. Prefer `\(...\)` for inline generated equations and `\[...\]` or `$$...$$` for display equations. Avoid relying only on `$...$` because dollar signs in normal text can be parsed as math.
5. In a sandboxed WebView, treat external scripts as dependencies that can fail when the user has no network or the environment blocks CDNs. Use a major-version-pinned or exact-version-pinned CDN for prototypes and consider bundled/hosted assets for production.
6. Keep text values escaped through Jinja or `html.escape`. Only pass equation strings generated by trusted app code into MathJax.
7. Add CSS for long equations: `overflow-wrap`, `overflow-x: auto`, and `max-width: 100%` on equation containers and `mjx-container`.
8. If JavaScript inserts or replaces equation HTML after initial page load, call `MathJax.typesetPromise([container])` after updating the DOM.

## Template Rules

- Put the MathJax config and script in `<head>`.
- Use a loading class if the raw TeX flash is distracting; reveal the report after MathJax startup resolves.
- Use Jinja loops for repeated wall/calculation blocks, but keep equation strings already formatted in Python.
- For VIKTOR report templates like `app/report/templates/wind_summary.html`, render with `WebResult(html=rendered_html)` after building the report view model in Python.
