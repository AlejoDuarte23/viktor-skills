# VIKTOR SCIA Worker Reference

## Scope

Use this reference when a VIKTOR app needs to build and run a SCIA Engineer model through the VIKTOR SCIA worker integration.

The standard architecture is:

1. The VIKTOR controller builds a `vkt.scia.Model`.
2. The controller creates supported SCIA objects with `model.create_*` methods.
3. `model.generate_xml_input()` returns the SCIA XML input and `.def` file.
4. `vkt.scia.SciaAnalysis` sends the XML, `.def`, and `.esa` template to the SCIA worker.
5. The worker runs SCIA Engineer on the connected Windows machine.
6. The controller reads output XML, updated `.esa`, or report files.
7. The app parses and displays results with VIKTOR views.

Official references:

- [SCIA Engineer integration guide](https://docs.viktor.ai/docs/create-apps/software-integrations/scia/)
- [SCIA Engineer SDK reference](https://docs.viktor.ai/sdk/api/external/scia/)
- [SCIA Engineer tutorial](https://docs.viktor.ai/docs/tutorials/integrate-scia/)

## Configuration

`viktor.config.toml` must expose the SCIA worker integration:

```toml
app_type = "simple"
python_version = "3.12"

worker_integrations = [
    "scia",
]
```

The official guide lists SCIA Engineer v19 through v25 as tested versions.

## ESA Template Requirements

The `.esa` file is not generated by VIKTOR. Create it in SCIA Engineer and use it as the template for `SciaAnalysis`.

Required checks:

- The SCIA version used to create the `.esa` template must match the version run by the worker.
- Materials referenced from VIKTOR must already exist in the `.esa` template.
- Material names and SCIA material ids must match between the VIKTOR model and the `.esa` file.
- The `.esa` file must define an XML I/O document named `output`, unless a different `xml_doc_name` is passed to `SciaAnalysis`.
- When manually importing generated XML into SCIA, keep the generated `.xml` and `.def` files in the same folder.

## Testing Notes

Do not run a real SCIA worker in unit tests.

Mock these behaviors:

- `Model.generate_xml_input()`
- `SciaAnalysis.execute()`
- `SciaAnalysis.get_xml_output_file()`
- `SciaAnalysis.get_updated_esa_model()`
- `SciaAnalysis.get_engineering_report()`

Use `viktor.testing.mock_SciaAnalysis` when the test only needs deterministic output files from the worker methods.
